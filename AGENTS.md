# Правила работы над проектом

## Описание проекта

**2030ai-claudecode-allecosystem-sync** — исходник инструкции `ecosystem-sync`. Синхронизирует скиллы, MCP-серверы и agent context между Claude Code CLI, Cursor и Codex App/CLI. Claude Code — источник истины; Cursor и Codex получают симлинки и аддитивные записи конфигов.

Основной сценарий: пользователь даёт Codex App ссылку на репозиторий и просит выполнить `ecosystem-sync` (см. README.md и PROMPT.md). Установка как локальный skill в `~/.claude/skills/ecosystem-sync` остаётся дополнительным вариантом.

Проект намеренно prompt/skill-first: не превращать его в обязательный deterministic CLI. Скрипты допустимы как read-only guardrails или узкие валидаторы; конкретные миграционные действия выбирает агент по контексту пользователя и safety rules.

## Связанные ресурсы

- **SKILL.md** — сам скилл (на английском, для AI-агентов). Единственная точка входа в логику.
- **README.md** — для людей (русский), основной one-prompt запуск.
- **PROMPT.md** — короткий copy-paste промпт для Codex App.
- **references/** — platform-matrix, mcp-format-guide, native-skills-registry, troubleshooting.

## Правила разработки скилла

- **Safety rules из SKILL.md — железные гарантии**, не смягчать их правками:
  - ADDITIVE ONLY: скилл не удаляет файлы, симлинки, записи конфигов.
  - Токены — никогда не логировать, не выводить, не писать (показывать как `<TOKEN>`).
  - `~/.codex/auth.json` не трогать никогда.
  - Нативные скиллы платформ не перезаписывать.
- Claude slash commands не являются целевым форматом миграции: если поведение нужно сохранить, переносить его в SKILL.md, а не создавать command shims.
- Языковой режим: SKILL.md и `references/` — на английском. README.md — на русском.
- Изменения в логике синка — сначала обновлять `references/platform-matrix.md`, если появилась новая платформа или изменился путь.
- Перед коммитом — прогон read-only audit/dry-run по `SKILL.md` или установленному `/ecosystem-sync audit`: инструкция должна отработать без записи на чистой установке.

## CRITICAL: Использовать встроенные инструменты вместо Bash-команд

- **Read** вместо `cat`, `head`, `tail` в Bash — для чтения файлов
- **Glob** вместо `ls`, `find` в Bash — для поиска файлов по паттерну
- **Grep** вместо `grep`, `rg` в Bash — для поиска по содержимому
- **Edit** вместо `sed`, `awk` в Bash — для редактирования файлов
- **Write** вместо `echo >`, `cat <<EOF` в Bash — для записи новых файлов

Bash использовать только для команд, которые нельзя выполнить встроенными инструментами (git-операции, CLI-инструменты).

## Чек-лист перед завершением задачи

- `/ecosystem-sync audit` проходит без ошибок.
- Safety rules не ослаблены.
- Если добавлена новая платформа/путь — обновлён `references/platform-matrix.md`.
- README.md и SKILL.md согласованы по поведению скилла.
